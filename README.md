## 🔍 Обзор проекта

**«ИИ-ассистент для Консультант по дизайн-проектам»** — это интеллектуальный чат-бот, созданный для автоматизации первичного взаимодействия с клиентами в сфере дизайн-услуг. Ассистент ведёт осмысленный диалог, отвечает на вопросы по услугам и тарифам, а также записывает клиентов на консультацию с автоматической синхронизацией в Google Календарь.

### Ключевые преимущества

| Преимущество | Описание |
|-------------|----------|
| 🤝 **Естественный диалог** | Нейросеть поддерживает контекст беседы, адаптируется под стиль общения клиента |
| 📚 **Экспертные ответы** | База знаний на основе материалов студии обеспечивает точность ответов до 95% [[20]] |
| 🗓️ **Автоматизация записи** | Интеграция с Google Calendar позволяет проверять слоты и бронировать встречи без участия человека [[20]] |
| ⏰ **Работа 24/7** | Бот отвечает на запросы клиентов в любое время, включая выходные и праздники [[20]] |
| 💰 **Снижение затрат** | Автоматизация рутинных запросов сокращает нагрузку на живых консультантов |

## 🎯 Цели и задачи

### Бизнес-цели
- Увеличить конверсию первичных обращений в записанные консультации на **15-20%**
- Сократить время реакции на запрос клиента до **менее 30 секунд**
- Освободить до **40% рабочего времени** консультантов от рутинных вопросов
- Обеспечить единый стандарт качества ответов для всех клиентов

### Технические задачи
- Настроить осмысленное ведение диалога с сохранением контекста
- Интегрировать базу знаний по услугам, тарифам и процессам студии
- Реализовать запись на встречу с проверкой доступности в реальном времени
- Обеспечить безопасную передачу данных и соответствие требованиям конфиденциальности

## ⚙️ Функциональные требования

### 1. Осмысленное ведение диалога

```yaml
Диалоговый движок:
  - Поддержка многоходового диалога с сохранением контекста
  - Распознавание намерений (intent recognition):
      * запрос информации об услугах
      * уточнение стоимости и сроков
      * запись на консультацию
      * вопрос по портфолио
      * техническая поддержка
  - Адаптация стиля общения под клиента (формальный/дружелюбный)
  - Обработка уточняющих вопросов и возражений
  - Плавная передача диалога человеку при сложных запросах
```

### 2. Ответы на вопросы по базе знаний

```yaml
База знаний:
  - Структурированные данные по услугам:
      * типы дизайн-проектов (интерьер, ландшафт, коммерция)
      * этапы работы и сроки
      * ценовые пакеты и условия оплаты
      * примеры работ и кейсы
  - Частые вопросы (FAQ) с вариативными формулировками
  - Динамическое обновление контента без переобучения модели
  - Приоритизация ответов: официальные источники > обновлённые данные
```

### 3. Запись клиента на встречу

```yaml
Сценарий записи:
  1. Сбор контактных данных (имя, телефон, email)
  2. Уточнение предпочтений:
      * тип консультации (онлайн/офлайн)
      * желаемая дата и время
      * тема обсуждения
  3. Проверка доступности через Google Calendar API
  4. Предложение альтернативных слотов при конфликте
  5. Подтверждение записи и отправка уведомления клиенту
  6. Автоматическое создание события в календаре консультанта
  7. Напоминание о встрече за 24 часа (опционально)
```

- Поддержка длинного контекста для анализа сложных запросов [[8]]
- Высокая точность следования инструкциям (instruction-following) [[1]]
- Возможность тонкой настройки стиля ответов через system prompt [[4]]
- Поддержка русского языка на уровне носителя

**Пример структуры системного промпта:**
```markdown
Ты — профессиональный консультант студии дизайна интерьеров.

и записывать на консультацию.

Правила общения:
1. Будь дружелюбным, но профессиональным
2. Не придумывай информацию — если ответа нет в базе, предложи связаться с менеджером
3. При записи на встречу обязательно уточни: имя, контакт, предпочтительное время
4. Используй эмодзи умеренно для улучшения восприятия
```

## ⚙️ Настройка системного промпта

### Базовая структура промпта для Qwen

```markdown
# РОЛЬ
Ты — ИИ-ассистент студии дизайна «Консультант по дизайн-проектам». 
Твоя миссия: помогать клиентам быстро получать информацию и записываться на консультацию.

# СТИЛЬ ОБЩЕНИЯ
- Тон: профессиональный, но тёплый и дружелюбный
- Язык: русский, без сложных терминов (объясняй при необходимости)
- Эмодзи: используй умеренно (1-2 на сообщение) для улучшения восприятия
- Структура: разбивай длинные ответы на пункты с маркерами

# ПРАВИЛА РАБОТЫ С ИНФОРМАЦИЕЙ
✅ Используй ТОЛЬКО данные из подключённой базы знаний
✅ Если информации нет — честно скажи и предложи связаться с менеджером
✅ Не придумывай цены, сроки или условия, которых нет в базе
✅ При цитировании указывай источник (например, "согласно нашему прайсу...")

# СЦЕНАРИЙ ЗАПИСИ НА ВСТРЕЧУ
1. Собери: имя, контакт, формат (онлайн/офлайн), предпочтительные даты
2. Через интеграцию проверь доступность в Google Calendar
3. Предложи 2-3 варианта слотов
4. После подтверждения — создай событие и отправь детали клиенту
5. Зафиксируй в логе: [ЗАПИСЬ] Клиент: {имя}, Дата: {дата}, Статус: подтверждено

# ОБРАБОТКА ВОЗРАЖЕНИЙ
- "Дорого" → объясни ценность, предложи базовый пакет или рассрочку
- "Долго" → уточни срочность, предложи экспресс-формат
- "Не уверен(а)" → предложи бесплатную 15-минутную консультацию для знакомства

# ЭСКАЛАЦИЯ
Передай диалог человеку, если:
• Клиент просит индивидуальное коммерческое предложение
• Возникает конфликтная ситуация или жалоба
• Запрос выходит за рамки компетенции бота

# БЕЗОПАСНОСТЬ
- Не запрашивай и не храни платежные данные
- Не передавай персональные данные третьим лицам
- При запросе на удаление данных — направь к менеджеру по конфиденциальности
```

### Рекомендации по оптимизации промпта [[1]][[4]]
- Разделяй системные инструкции и пользовательский контекст
- Используй чёткие маркеры для разных типов задач (например, `#ЗАПИСЬ`, `#ОТВЕТ`)
- Добавляй примеры желаемых ответов для сложных сценариев (few-shot prompting)
- Регулярно тестируй промпт на пограничных случаях и обновляй правила

## 📊 Метрики эффективности

### Количественные показатели

| Метрика | Целевое значение | Метод измерения |
|---------|-----------------|----------------|
| **Точность ответов** | ≥ 95% | Выборочная проверка диалогов + фидбек клиентов [[20]] |
| **Конверсия в запись** | +15-20% к базовой | Сравнение периодов до/после внедрения |
| **Среднее время ответа** | < 30 секунд | Логи платформы Suvvy |
| **Доля решённых запросов без участия человека** | ≥ 70% | Аналитика эскалаций в панели управления |
| **Удовлетворённость клиентов (CSAT)** | ≥ 4.5/5 | Пост-диалоговый опрос (опционально) |

### Качественные индикаторы
- Снижение количества повторных вопросов по одной теме
